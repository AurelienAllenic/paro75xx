Welcome

Welcome to our Legendary one page html template thank you for purchasing one of our products, your support is much appreciated.

Please drop us a line and let us know what you think of this template, if you have any suggestions for improvements or extra features, if there’s something you don’t like or you just want to say “Hello”.

We count on your help to constantly improve the quality of our products and bring more value to you and all of our other customers.

Now, let’s get to the help part.

First of all, customization of this template will, of course, require knowledge of HTML and for heavier customization a bit of CSS and Javascript.

Getting started

If you have already expirence in web design and template editing / uploading you can skip this section.

Requirements

Excluding the language skills mentioned before you need proper software to edit the template files.

You can find a recommendation for sofware in the support section at the bottom of this page.

You also need a hosting service where you can upload your files to the internet.

Installation

This a very generic guide for installing / uploading a html template.

If your hosting provider is providing a guide how to upload a html website / template you should use it.

  1. Get your files

    Open the HTML files folder that is including your main template files.

  2. Log In

    Log into your hosting space via an FTP software ( Filezilla or Cyberduck for example ).

  3. Upload

    Upload all files from your html files folder into another folder on your hosting space via ftp software.

Essentials

Layout

You can find the javascript code used for the layout / core in your assets/js/script.js file and you can find the css used for the layout / core in your assets/css/layout.css file.

Loader

The logo that you can view as long as the page is loading is located inside your assets/images folder and is called loading-logo.png.

Navigation

If you want to change the navigation you can do this simply in your config.js file in your main directory under the section /** 2. MAIN CONFIG. You can choose between fixed, popup or sticky navigation.

Skins

Color skins can be changed inside your head section of your index html document you have choosen ( index-1.html, index-2.html, index-3.html, ... )

Simply find the color stylesheet link at the bottom of the head section of your html file and update the file path href="assets/css/color/default.css". Replace the default.css with one of the colors below.

  • amber.css
  • blue.css
  • blue-grey.css
  • brown.css
  • cyan.css
  • deep-orange.css
  • deep-purple.css
  • green.css
  • indigo.css
  • light-blue.css
  • light-green.css
  • lime.css
  • orange.css
  • pink.css
  • purple.css
  • red.css
  • teal.css
  • yellow.css

If you want to use another color simply use the .custom.css or .custom.scss stylesheets inside your assets/css/color folder.

Parallax

You can disable the parallax scrolling effect in your config.js file in your main directory under the section
/** 2. MAIN CONFIG.

If you want to customize the parallax effect you can use the official Parallaxer documentation.

Animations

You can simply add or remove the scrolling animation by adding or removing the scroll-animated-from-bottom class.

And you can also customize the animation in your assets/js/script.js file just find the following code and edit it with the offical ScrollReveal Documentation.

if (option_animate_content_on_scroll == "on") {
            
            window.sr = ScrollReveal();
            sr.reveal(".scroll-animated-from-bottom", { 
                duration: 500,
                delay: 0,
                origin: "bottom",
                rotate: { x: 0, y: 0, z: 0 },
                opacity: 0,
                distance: "80px",
                viewFactor: 0.3,
                scale: 1,
            });
        
    }

Popups

Notice!

All Popups are using ajax. They will not work on your local machine without a web server extension for your browser or a local server environment ( MAMP, WAMP, XAMP, ... ).

You can see all available popups on your shortcodes page and all popup ajax files are located inside your pages/ajax/popups/ folder.

Every popup / lightbox can be used at any place or element just wrap your choosen element with the <a></a> tags.

Use this for showcase or news popups :
<a class="cbp-singlePage" href="pages/ajax/popups/showcases/showcase-popup-full-page-image.html" ></a>

Use this for lightbox popups :
<a class="cbp-lightbox" href="assets/images/popup/showcase/1.jpg" ></a>

Hero

You can find the javascript code used for the hero section in your assets/js/hero.js file and you can find the css used for the hero section in your assets/css/hero.css file.

You can disable or enable the following global hero options inside section /** 3. HERO CONFIG in your config.js file:

  • Down scroll animation effect
  • 3D hover effect
  • Parallax hover effect
  • Particle background effect

If you want to use the slider revolution hero index-5.html you can find the official slider revolution documentation here.

Below you find a how to guide for each hero background available for index-1.html, index-2.html, index-3.html and index-4.html.

Image

If you want to use the image background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "image" at first.

After this you can simply overwrite 1.jpg in your assets/images folder or you can jump to section /** 3.1 HERO CONFIG [ IMAGE BACKGROUND ] in your config.js file and update the image path or enter a url.

Slider

If you want to use the slider background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "slider" at first.

After this you can simply overwrite 1.jpg, 2.jpg, 3.jpg in your assets/images folder or you can jump to section /** 3.2 HERO CONFIG [ SLIDER BACKGROUND ] in your config.js file and update the image paths or enter urls.

Optional you can also update the slide delay, transition and transition duration in this particular section in your config.js file.

KenBurns

If you want to use the KenBurns slider background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "kenburns" at first.

After this you can simply overwrite 1.jpg, 2.jpg, 3.jpg in your assets/images folder or you can jump to section /** 3.3 HERO CONFIG [ KENBURNS BACKGROUND ] in your config.js file and update the image paths or enter urls.

Optional you can also update the slide delay, transition and transition duration in this particular section in your config.js file.

Youtube

Notice!

The youtube background option doesn't work as background for mobile devices due to the restriction policy adopted by all on managing multimedia files via javascript. The default fallback is the image background for mobile devices.

If you want to use the YouTube video background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "youtube" at first.

After this you can simply jump to section /** 3.4 HERO CONFIG [ YOUTUBE BACKGROUND ] in your config.js file and update the youtube url
( Do not use embed urls! ).

Optional you can also edit the start point, end point and mute the video from the begining in this particular section in your config.js file.

Color

If you want to use the solid color background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "color" at first.

After this you can simply jump to section /** 3.5 HERO CONFIG [ COLOR BACKGROUND ] in your config.js file and enter a custom css color value.

Gradient

If you want to use the gradient color background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "gradient" at first.

Additonal you can update the color matrix and transition duration in section /** 3.6 HERO CONFIG [ GRADIENT BACKGROUND ] in your config.js file.

Sphere

If you want to use the canvas sphere background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "sphere" at first.

Additonal you can update the object distance in section /** 3.7 HERO CONFIG [ SPHERE BACKGROUND ] in your config.js file.

Waves

If you want to use the canvas waves background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "waves" at first.

Additonal you can update the object distance, dot spacing, dot amount on x axis and dot amount on y axis in section
/** 3.8 HERO CONFIG [ WAVES BACKGROUND ]in your config.js file.

Mesh

If you want to use the canvas mesh background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "mesh" at first.

Additonal you can update the mesh color and mesh spotlight size in section
/** 3.9 HERO CONFIG [ MESH BACKGROUND ] in your config.js file.

Space

If you want to use the canvas space background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "space" at first.

Additonal you can update the star amount and star speed in section
/** 3.10 HERO CONFIG [ SPACE BACKGROUND ] in your config.js file.

Custom

If you want to use a custom background you have to update the background option in your config.js file under section /** 3. HERO CONFIG to "custom" at first.

After this you can drop your code in section
/** 3.11 HERO CONFIG [ CUSTOM BACKGROUND ] in your config.js file.

Make sure to use $("#canvas") as selector for your code.

Sections

You can find the javascript code used for all sections and sections content in your assets/js/blocks.js file and you can find the css used for the hero section in your assets/css/blocks.css file.

Every section can have a black, white or parallax background.

If you want to make a section black simply find the div element with the content class and remove the bg-white class ( when given ) and add the bg-black class to this specific div.

If you want a section in white simply repeat the step above with inverted classes ( remove class bg-black add class bg-white ).

If you want a parallax section find the div element with the content class and add the bg-black and bg-parallax class to this specific div ( remove the bg-white when given ). After this add the parallax container ( copy it from service, process, news or footer section ) with the dzsparallaxer class inside your div element with the content class.

About

If you want a custom and unique looking about section you can simply do this by adding text or images and if you want a custom layout you can rearrange the elements by using the bootstrap grid.

Service

If you need another amount of services you can simply remove or add service items inside your html document but make sure to update the % width in your assets/css/blocks.css file on line 439 ( .block-service-1 .tab-navigation li ).

You can also rearrange the text below the service items by using the bootstrap grid.

Work

The work section is based on the cube portfolio plugin portfolio module. You can customize it by updating the code in your assets/js/blocks.js file under section /** 1. WORK and the help from cube portfolio documentation.

If you want to add more elements to the load more button you can do this by simply adding items to your pages/ajax/loadMore.html html document.

If you want to add showcase popups make sure to create new ajax html files ( dublicate and rename popup html files ) and link to them. Jump to popup documentation.

If you want to add more popups lightboxes you just have to link to your source files / urls ( image, youtube, vimeo, mp4 or Soundcloud ) no html files requiered. Jump to popup documentation.

Process

The process section is based on the cube portfolio plugin slider module. You can customize it by updating the code in your assets/js/blocks.js file under section /** 2. PROCESS and the help from cube portfolio documentation.

Team

The team section is based on the cube portfolio plugin slider module. You can customize it by updating the code in your assets/js/blocks.js file under section /** 3. TEAM and the help from cube portfolio documentation.

You can rearrange everything below the team slider by using the bootstrap grid.

News

The news section is based on the cube portfolio plugin slider module. You can customize it by updating the code in your assets/js/blocks.js file under section /** 4. NEWS and the help from cube portfolio documentation.

If you want to add news popups make sure to create new ajax html files ( dublicate and rename popup html files ) and link to them. Jump to popup documentation.

Contact

Notice!

The contact form is not work on your local machine.( html file loaded directly into browser )

The contact form is ajax and php based make sure that your hosting package includes a running mail server.

Make sure to enter your email in your assets/contact.php file.

Notice!

You need a Google Maps API key to use the Google Map. Get it here! Otherwise, the map will not show up and return an error.

You can insert your Google Maps API key in your config.js file. Simply find the /** 2. MAIN CONFIG section and overwrite the key from the option_google_maps_api_key variable.

Directly below this setting you can modify the map cordinates. The variable is called option_google_maps_coordinates. Make sure that you just change the numbers.

Footer

If you want to replace the footer logo you can simply do this by overwriting the assets/images/logo-white-footer.png image file.

The footer layout can be rearranged by using the bootstrap grid.
